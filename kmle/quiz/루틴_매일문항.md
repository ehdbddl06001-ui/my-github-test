# 매일 오전 6시 새 문항 생성 루틴 (Claude Code 루틴)

매일 **오전 6시**에 Claude Code 루틴이 세션을 깨워 **새 KMLE 문항 1개**를 만들고 저장소에 커밋·푸시합니다.
**Anthropic API 키를 직접 발급/관리하지 않습니다** — Claude Code의 스케줄 루틴(웹앱 자동화)이 그 일을 합니다.

## 1. 스케줄 설정 위치 (한 번만)

Claude Code 웹앱에서 이 저장소에 대한 **자동화(Automation/Schedule)** 를 만듭니다.
- 실행 주기: **매일 06:00 (Asia/Seoul), 반복**
- 대상 브랜치: **main** (이미 병합되어 사용 중이므로. 원하면 별도 브랜치+PR로도 가능)
- 실행 지시(프롬프트): 아래 **2번 프롬프트**를 그대로 붙여넣기

> 참고: 세션 내부 스케줄러(CronCreate)는 컨테이너가 회수되면 사라지므로 매일 실행에 부적합합니다. 반드시 웹앱 자동화로 등록하세요.

## 2. 루틴에 넣을 지시 프롬프트 (복사해서 사용)

```
너는 KMLE 문항 생성 루틴이다. 아래 규칙에 따라 오늘자 새 문항 1개를 만들어 저장소에 반영하라.

[대상] kmle/quiz/questions/<NN_과목>.json 에 문항 1개를 추가한다.

[과목 로테이션] 오늘 날짜(요일 또는 일자)를 기준으로 16개 과목을 순환 선택한다.
 예: (일자 % 16)로 과목 인덱스를 정한다. 최근 며칠간 나온 과목과 겹치지 않게 한다.

[중복 방지] 같은 과목의 기존 문항들과 핵심 주제/정답 포인트가 겹치지 않는 새 주제를 고른다.
 기존 문항의 vignette·question을 먼저 읽고, 다루지 않은 질환·개념을 선택한다.

[형식] 다음 필드를 모두 채운 JSON 객체 1개:
 - id: "gen-YYYYMMDD-01" (오늘 날짜, 여러 개면 -02 증가)
 - subject, subject_file(예: "02_순환기"), 
 - type: 텍스트로 풀리는 유형만 사용(▲ 증례-텍스트 / ● 진단기준 / ■ 기전개념 / ◆ 기출변형 / ◇ 통계해석).
   ★ 증례-이미지는 실제 이미지가 없으므로 절대 사용하지 않는다.
 - created: 오늘 날짜 "YYYY-MM-DD" (반드시 포함 — '오늘의 문항' 구역이 이 필드로 필터한다)
 - vignette: 검사수치·진찰소견을 포함한 상세 증례(6~10문장)
 - question, options(정확히 5개), answer(1~5 정수)
 - explanation: {진단, 정답근거, 오답감별, 임상핵심} 4개 키 모두
 - source: 대표 교과서/가이드라인

[자가 검증] 커밋 전 스스로 점검한다: 정답이 의학적으로 타당한가? 오답 4개가 명확히 오답인가?
 최신 가이드라인과 배치되지 않는가? 확신이 없으면 더 안전하고 표준적인 주제로 교체한다.

[반영] 
 1) 해당 과목 JSON에 추가
 2) `cd kmle/quiz && python3 quiz.py --export` 실행(문항/*.md 및 docs/questions.js 갱신)
 3) `git add -A && git commit` 후 main에 push
 커밋 메시지: "매일 문항 추가: <과목> (<id>)"

[안전] 생성 문항은 학습용이며 자동 생성이므로, 사용자가 주기적으로 검토한다는 전제로 만든다.
```

## 3. 학습 구역과의 연동 (이 부분이 사용자 요청의 핵심)

문항이 쌓여도 **어떤 문항을 풀지 골라서** 학습할 수 있도록, 퀴즈는 세 구역으로 나뉩니다.

| 구역 | 무엇을 푸나 | 필터 기준 |
|------|-------------|-----------|
| **오늘의 문항** | 오늘 루틴이 만든 새 문항만 | `created == 오늘 날짜` |
| **전체 문항** | 누적된 모든 문항(과목 선택 가능) | 전체/과목 |
| **오답 복습** | 오답노트에 쌓인 문항만 | 저장된 오답 id |

- **CLI**: `python3 quiz.py` 실행 시 메뉴에서 1/2/3 선택. 또는 `--today`, `--all`, `--review` 플래그.
- **웹**: 상단 "학습 구역" 드롭다운에서 선택.

이렇게 하면 "매일 6시 새 문항"은 **오늘의 문항** 구역에서 바로 풀 수 있고(기존 문항에 섞여 늦게 등장하지 않음), 복습하고 싶을 때만 **전체/오답** 구역을 고르면 됩니다.

## 4. id·날짜 규칙 요약
- 기존 32문항: `created` 없음 → 항상 "전체"에만 속함(오늘의 문항엔 안 나옴)
- 루틴 생성 문항: `id = gen-YYYYMMDD-NN`, `created = YYYY-MM-DD`
- "오늘의 문항"은 그날 `created`가 오늘인 것만 노출 → 다음 날이면 자동으로 "전체"로 편입

## 5. 주의
자동 생성 문항은 의학적 검증을 자동화한 것이 아니므로, **주기적으로 사람이 검토**하시길 권합니다.
오류를 발견하면 해당 JSON을 고치고 `python3 quiz.py --export`로 재생성하면 됩니다.
