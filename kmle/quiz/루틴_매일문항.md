# 매일 오전 6시 새 문항 생성 루틴 (Claude Code 루틴)

매일 **오전 6시**에 Claude Code 루틴이 세션을 깨워 **하루 32문항(16개 과목 × 2문항)**을 만들고 저장소에 커밋·푸시합니다.
**Anthropic API 키를 직접 발급/관리하지 않습니다** — Claude Code의 스케줄 루틴(웹앱 자동화)이 그 일을 합니다.

## 1. 스케줄 설정 위치 (한 번만)

Claude Code 웹앱에서 이 저장소에 대한 **자동화(Automation/Schedule)** 를 만듭니다.
- 실행 주기: **매일 06:00 (Asia/Seoul), 반복**
- 기준 브랜치: **main**(루틴은 매 실행마다 main을 clone한 뒤 `claude/`로 시작하는 새 브랜치를 만들어 그 위에서 작업합니다)
- 실행 지시(프롬프트): 아래 **2번 프롬프트**를 그대로 붙여넣기

> 참고: 세션 내부 스케줄러(CronCreate)는 컨테이너가 회수되면 사라지므로 매일 실행에 부적합합니다. 반드시 웹앱 자동화로 등록하세요.

> 주의: 루틴은 기본적으로 `claude/`로 시작하는 브랜치에만 push할 수 있고, main 같은 브랜치에는 직접 push할 수 없습니다(저장소 보호를 위한 기본 제한). 따라서 아래 프롬프트는 매번 새 `claude/` 브랜치에서 작업 후 **PR을 여는 방식**으로 되어 있습니다. main에 직접 push하도록 하려면 루틴 설정의 Permissions에서 "Allow unrestricted branch pushes"를 켜야 하지만, 그 경우 검토 없이 바로 main이 바뀌므로 권장하지 않습니다.
>
> 또한 루틴 실행이 "완료"로 표시되는 것은 세션이 에러 없이 끝났다는 뜻일 뿐, 프롬프트 작업(문항 생성·커밋·PR)이 실제로 성공했다는 뜻은 아닙니다. 매일 실행 후 claude.ai/code/routines에서 해당 run의 트랜스크립트나, GitHub의 새 PR을 직접 확인하세요.

## 2. 루틴에 넣을 지시 프롬프트 (복사해서 사용)

```
너는 KMLE 문항 생성 루틴이다. 오늘자 새 문항 32개(16개 과목 × 2문항)를 만들어 저장소에 반영하라.

[시작] main을 기준으로 `claude/kmle-daily-YYYYMMDD`(오늘 날짜) 새 브랜치를 만들고 그 위에서 작업한다.
 (이 루틴은 main에 직접 push할 권한이 없다. 반드시 `claude/`로 시작하는 브랜치를 사용한다.)

[0단계 — 중복 방지 목록 확보]
 먼저 `cd kmle/quiz && python3 quiz.py --recent-topics 14` 를 실행한다.
 출력된 '최근 14일 내 주제 목록'을 확보한다. 새로 만들 문항은 각 과목에서 이 목록과 겹치지 않아야 한다.
 (14일보다 오래된 주제는 다시 사용해도 된다.)

[대상] 16개 과목 각각에 2문항씩 추가한다. 파일: kmle/quiz/questions/<NN_과목>.json
 과목: 01_의학총론 02_순환기 03_호흡기 04_소화기 05_신장 06_내분비 07_혈액종양 08_감염
       09_류마티스 10_알레르기 11_산부인과 12_소아청소년과 13_정신건강의학과 14_신경과
       15_기타마이너 16_의료관계법규

[중복 방지 규칙] 각 과목마다, 위 0단계의 최근 14일 주제 및 같은 과목 기존 문항의 핵심 주제와
 겹치지 않는 서로 다른 2개 주제를 고른다. 억지스러운 주제보다 임상적으로 의미 있는 고빈도 주제를 우선한다.

[난이도 규칙] 과목당 2문항 중 최소 1문항은 다음을 지킨다(가능하면 2문항 모두):
 - 다단계 추론: 소견에서 값을 계산하거나(예: CrCl, CHA2DS2-VASc, Child-Pugh, FENa, 산-염기 1차/보상 판정)
   그 중간 결론을 한 번 더 거쳐야 최종 정답에 도달하도록 설계한다. 소견 나열 → 바로 정답 선택은 지양한다.
 - 기초의학 연계: 임상 소견뿐 아니라 병태생리·약리기전·미생물학적 특성 등 기초의학 단서가 있어야
   감별이 가능하게 만든다. 임상 패턴 암기만으로 풀리지 않게 한다.
 - 정답 단어 비노출: vignette·question 어디에도 정답 진단명/핵심 병명을 문자 그대로 쓰지 않는다.
   (예: "조증삽화"라고 쓰지 말고, 그 진단에 해당하는 행동·증상을 6~10문장으로 서술해 유추하게 한다.)
   options의 다른 오답 보기에는 병명이 등장해도 무방하다.

[각 문항 형식] 다음 필드를 모두 채운 JSON 객체:
 - id: "gen-YYYYMMDD-NN" (오늘 날짜, NN은 01부터 32까지 증가, 파일 전체에서 유일)
 - subject, subject_file(예: "02_순환기")
 - type: 텍스트로 풀리는 유형만(▲ 증례-텍스트 / ● 진단기준 / ■ 기전개념 / ◆ 기출변형 / ◇ 통계해석).
   ★ 증례-이미지는 실제 이미지가 없으므로 절대 사용하지 않는다.
 - created: 오늘 날짜 "YYYY-MM-DD" (필수 — '오늘의 문항' 구역이 이 필드로 필터한다)
 - topic: 핵심 주제를 나타내는 짧은 한 줄(예: "심방세동 뇌졸중 예방(CHA2DS2-VASc)").
   이 값이 중복 판정(최근 14일)의 기준이 되므로 반드시 넣는다.
 - vignette: 검사수치·진찰소견을 포함한 상세 증례(6~10문장). 위 [난이도 규칙]을 반영한다.
 - question, options(정확히 5개), answer(1~5 정수)
 - explanation: {진단, 정답근거, 오답감별, 임상핵심} 4개 키 모두
 - source: 대표 교과서/가이드라인
 - appendix: {가이드라인, 최신지견, 참고문헌} — 해설 아래 붙는 심화 부록(아래 [부록 작성] 참고)

[부록 작성] appendix는 다음 3개 키를 모두 채운다:
 - 가이드라인: 관련 진단/치료 기준을 표 형태 텍스트로 요약. 마크다운 표 문법 사용:
   "| 항목 | 기준 |\n|---|---|\n| ... | ... |" 처럼 개행(\n)과 파이프(|)로 작성한다(실제 이미지·그래프 파일은 만들지 않는다).
 - 최신지견: 해당 주제의 최근 개정 가이드라인·변경된 진료 지침이나 최신 연구 동향을 2~4문장으로 요약.
 - 참고문헌: 대표 문헌·가이드라인 2~3개를 문자열 배열로(예: ["Harrison's Principles of Internal Medicine, 21st ed.", "2023 ESC Guidelines for the management of..."]).

[자가 검증] 커밋 전 각 문항을 점검한다: 정답이 의학적으로 타당한가? 오답 4개가 명확히 오답인가?
 최신 가이드라인과 배치되지 않는가? topic이 최근 14일 목록과 겹치지 않는가?
 [난이도 규칙] 3가지(다단계 추론/기초의학 연계/정답 단어 비노출)를 과목당 최소 1문항이 만족하는가?
 appendix 3개 키가 모두 채워져 있고 가이드라인 표가 실제로 표 형식인가?
 확신이 없으면 더 안전하고 표준적인 주제로 교체한다.

[반영]
 1) 각 과목 JSON에 2문항씩 추가(총 32문항)
 2) `cd kmle/quiz && python3 quiz.py --export` 실행(문항/*.md 및 docs/questions.js 갱신)
 3) `git add -A && git commit` (커밋 메시지: "매일 문항 추가: 16과목 × 2문항 (YYYY-MM-DD)")
 4) `git push -u origin claude/kmle-daily-YYYYMMDD`
 5) main을 base로, 방금 만든 브랜치를 head로 하는 PR을 연다.
    제목: "매일 문항 추가: 16과목 × 2문항 (YYYY-MM-DD)"
    본문: 추가한 32문항의 과목·주제(topic) 목록을 표나 리스트로 요약
 6) PR은 자동 병합하지 않는다(사용자가 검토 후 병합). PR 링크를 세션 결과에 남긴다.

[부하 주의] 한 세션에서 32문항 생성이 무거우면 과목을 몇 개씩 나눠 순서대로 처리하되,
 마지막에 한 번만 --export 하고 커밋한다.

[안전] 생성 문항은 학습용이며 자동 생성이므로, 사용자가 주기적으로 검토한다는 전제로 만든다.
```

## 3. 중복 방지: "최근 14일"만 확인 (C)

- 각 문항의 `topic` 필드가 중복 판정의 키입니다.
- `python3 quiz.py --recent-topics [일수]` 는 **해당 기간(기본 14일) 내 생성된 문항의 주제**를 과목별로 출력합니다.
  ```
  # 최근 14일(2026-06-17 이후) 생성된 문항 주제 (이 목록과 겹치지 않게 생성하세요)
  [순환기]
    - (2026-07-01) 심방세동 뇌졸중 예방(CHA2DS2-VASc, DOAC)
  ...
  ```
- 루틴은 이 목록과만 겹치지 않으면 됩니다. **14일이 지난 주제는 다시 출제되어도 무방**합니다.
- 즉 소재 고갈 걱정 없이, 2주 주기로 핵심 주제가 자연스럽게 순환합니다.

## 4. 학습 구역 연동 (오늘의 문항 / 전체 / 오답 복습)

문항이 매일 32개씩 쌓여도 골라서 풀 수 있습니다.

| 구역 | 무엇을 푸나 | 필터 |
|------|-------------|------|
| **오늘의 문항** | 오늘 만든 32문항 (과목 선택 가능) | `created == 오늘` (+ 과목) |
| **전체 문항** | 누적 전체(과목 선택) | 전체/과목 |
| **오답 복습** | 오답노트에 쌓인 것 | 저장된 오답 id |

- **오늘의 문항은 과목 필터가 있어**, 32개를 과목별(각 2문항)로 골라 풀 수 있습니다.
- **CLI**: `python3 quiz.py` 메뉴에서 1(오늘) 선택 → 과목 선택. 또는 `--today [--subject 순환기]`.
- **웹**: 상단 "학습 구역" = 오늘의 문항 선택 → 과목 드롭다운에서 선택.

## 5. id·날짜·topic 규칙 요약
- 기존 32문항: `created` 없음 → 항상 "전체"에만
- 루틴 생성 문항: `id = gen-YYYYMMDD-NN`(01~32), `created = YYYY-MM-DD`, `topic = 주제 한 줄`
- "오늘의 문항"은 그날 `created`가 오늘인 것만 노출 → 다음 날이면 자동으로 "전체"로 편입

## 6. 주의
자동 생성 문항은 의학적 검증을 자동화한 것이 아니므로, **주기적으로 사람이 검토**하시길 권합니다.
오류를 발견하면 해당 JSON을 고치고 `python3 quiz.py --export`로 재생성하면 됩니다.

## 7. 난이도 규칙과 해설 부록

과목당 최소 1문항은 다음 3가지를 지키도록 프롬프트에 반영되어 있습니다.

1. **다단계 추론**: 소견에서 수치를 계산하거나 중간 판정을 거쳐야 정답에 도달하는 구조
2. **기초의학 연계**: 임상 패턴 암기가 아니라 병태생리·약리기전 등 기초 지식이 있어야 감별 가능
3. **정답 단어 비노출**: vignette/question에 정답 진단명을 그대로 쓰지 않고 증상·행동으로 서술

또한 각 문항의 해설(`explanation`) 아래에 심화 학습용 **부록(`appendix`)**이 붙습니다: 관련 가이드라인 기준표(마크다운 표), 최신 지견/개정 사항, 참고문헌 2~3개. CLI(`quiz.py`)와 웹 퀴즈(`docs/`) 모두 해설 다음에 부록을 렌더링합니다.

> 실제 이미지·그래프 파일은 생성하지 않습니다(근거: 위 3번 논의 — AI가 그린 의료 영상은 부정확할 위험이 있고, 실제 이미지를 가져오면 저작권·오진 검증 문제가 있습니다). "그래프"가 필요한 자료는 마크다운 표 형태의 텍스트로 대체합니다.
