# KMLE/USMLE 문제 생성 규칙

## 필드
`schemas/frontmatter.md` 의 문제형 규격을 그대로 따른다. 특히:
- `answer_separated: true` 필수. stem에는 정답 힌트를 넣지 않는다.
- `id` 는 `state.next_id('kmle')` 또는 `next_id('usmle')` 로만 발급.
- `topic` 은 16개 임상 과목 기준(내과/외과/소아/산부인과/…)에서 고른다.

## 저장 경로
- KMLE → `content/kmle/{연도}/{id}.md`
- USMLE → `content/usmle/{id}.md`

## USMLE 전용 규칙 (반드시 지킬 것)
- frontmatter에 `step`("Step 1" 또는 "Step 2")과 `exam_subject`를 **반드시** 넣는다.
  없으면 `indexer.py --check`에서 실패하고 웹/CLI 퀴즈에 분류되지 않는다.
  - Step 1 과목: Pharmacology, Immunology, Biochemistry, Microbiology, Pathology, Physiology
  - Step 2 과목: Internal Medicine, Surgery, Neurology, Pediatrics, Obstetrics & Gynecology, Psychiatry
- stem·보기는 실제 시험 형식대로 **영문**, 해설은 한글(핵심 용어 영어 병기)로 쓴다.
- 본문 `## 정답 및 해설`은 `- **키**: 값` 불릿 구조를 유지한다
  (`export_usmle_web.py`가 이 텍스트를 파싱해 웹/CLI 해설로 사용).
- 생성 후 반드시 `python pipelines/export_usmle_web.py`로 `docs/questions_usmle.js`를
  재생성해 같은 커밋에 포함한다(안 하면 개인 페이지에 새 문항이 안 뜬다).

## 품질 규칙
1. 최근 `recent_topics()` 로 나온 주제는 피한다(중복 방지).
2. 임상 추론형으로 작성: 단순 암기보다 감별진단·치료선택 위주.
3. 근거가 불확실하면 `confidence: medium/low` 로 낮추고 `source`에 근거를 남긴다.
4. 생성 직후 스스로 정답의 타당성을 한 번 검증한다. 의학적 정확도는
   자기검증에 한계가 있으므로, 애매하면 문제를 버리고 다시 만든다.

## 난이도 구조 (difficulty 필드 1~5) — 상세 기준
목표 분포: 하루 세트에서 **difficulty 3~5가 다수**가 되도록 한다(암기형 1~2는
과목당 최대 1개). `difficulty` 는 아래 정의에 맞춰 **정직하게** 매긴다.

| 값 | 인지 수준 | 판별 기준(이걸 만족해야 그 값) |
|----|-----------|------------------------------|
| 1 | 단순 회상 | 한 개 사실을 바로 떠올리면 풀림(정의·짝짓기). |
| 2 | 기본 적용 | 한 단계 적용. 전형적 제시 → 진단/약물 1:1. |
| 3 | 임상 추론 | 소견을 **종합**해 감별 후 최적 선택. 최소 1개 감별지 제거 필요. |
| 4 | 2차 추론 | "진단"이 아니라 **다음 단계·기전·왜**를 물음. 정답까지 **2단계 이상** 사고. |
| 5 | 함정·통합 | 흔한 오답이 매력적이거나, **하면 안 되는 것**(금기)·기초의학 통합을 요구. |

### 난이도를 올리는 5가지 구조적 장치 (difficulty ≥ 4 는 최소 2개 적용)
1. **2차 질문(second-order lead-in)**: "진단은?" 대신 "**다음 단계는?**", "이
   소견을 가장 잘 설명하는 **기전은?**", "**금기**인 것은?", "**시행하면 안 되는**
   처치는?" 형태로 묻는다. 진단명은 stem이 가리키되 **정답은 그 다음 사고**다.
2. **정답 단어 비노출**: stem·질문에 정답 보기의 핵심 단어(약물명·진단명)를 넣지
   않는다. 소견으로 유추하게 하고, 정답 힌트가 새면 문제를 다시 만든다.
3. **동질적·매력적 오답(homogeneous distractors)**: 오답은 모두 같은 범주(예: 전부
   "치료" 또는 전부 "기전")로 맞추고, **흔한 오개념**을 하나씩 심는다. 명백히 틀린
   장식성 보기는 금지. 각 오답이 왜 틀렸는지 해설의 **오답감별**에 letter로 단다.
4. **기초의학 연계(Step 1) / 함정(Step 2)**: Step 1 은 생리·생화·약리 기전으로
   정답을 설명하게 하고, Step 2 는 "정답처럼 보이지만 해가 되는 선택"(금기·순서
   오류)을 오답으로 배치한다.
5. **정답 위치 분산**: 한 세트에서 정답을 **A~E에 고르게** 분산한다(모두 A 같은
   위치 편향 금지 — 앱은 보기 순서를 셔플하지 않는다). "위/아래 모두" 류 보기 금지.

### 커밋 전 문항 자가 점검(체크리스트)
- [ ] 질문이 회상이 아니라 **적용/2차 추론**을 요구하는가(difficulty ≥ 3)?
- [ ] stem에 정답을 직접 알려주는 단어가 없는가(`answer_separated` 원칙)?
- [ ] 오답 5개가 **동질적**이고 각각 **그럴듯한 오개념**인가?
- [ ] 정답 위치가 세트 안에서 특정 letter로 쏠리지 않았는가?
- [ ] 해설에 **정답근거 + 오답감별(letter별) + 임상핵심 + 출처**가 다 있는가?
- [ ] 의학적으로 논쟁적이면 `confidence` 를 낮추고 근거를 남겼는가?

## 본문 구조(사람이 읽는 부분)
```
## 문제
{stem}

- A. …
- B. …

## 정답 및 해설
> 정답: {answer}
{explanation}
```
